# Design Specification - Hackathon & Judge Guide in READMEs

This design specification details adding hackathon context, pitch goals, and the interactive judge path to both repository README files.

## Objective
To make the codebase immediately accessible and navigable for hackathon judges, highlighting the pitch strategy, demo path, and project assets.

## Proposed Layout Changes

### 1. Root README (`README.md`)
We will restructure the root README to lead with:
- Centered KapitBizPH logo (already approved).
- **Pitch & Hackathon Context**: Detailing that the project targets the *Resilience* theme (lot drawn on 2026-07-15) and focuses on protecting MSME inventory during local disasters.
- **Judge / Demo Path**: Step-by-step click guide to run through the transaction demo.
- **Repository Layout Map**: Linking the `web/` app code, `pitch/` slide decks, and `planning/` assets.
- **Project Boundaries**: Clearly detailing simulated feeds vs local client logic.

### 2. Web README (`web/README.md`)
We will sync these details to ensure the `web/README.md` is also fully aligned and presents the same structured pitch goals and judge path at the top.

## Verification Plan
1. Preview both README files to ensure correct link formatting and clean readability.
2. Confirm the relative paths/links between files inside the repo resolve properly.
